AWS 서버리스로 서버 고민 없이 웹 애플리케이션 구축하기

서버리스란? 사용자가 관리할 서버가 없다 -> 서버의 관리는 AWS가 함 -> 개발 팀은 운영작업에 리소스를 투입 안해도 되고, 코드 작성에 집중 가능

가상머신의 한계 = host os 위에 guest os를 올리다보니 무겁고 배포가 오래걸림 -> 별도의 게스트 Os를 올리지 않는 컨테이너 기술 사용하게 됨

서버리스의 장점 : 서버 관리 필요 없음, 사용한 만큼만 지불하여 불필요한 비용 발생 x -> 특정 시간에 트래픽이 몰릴 경우 효율적, 자동 스케일링, 보안
-> 인프라 관리가 아닌 비즈니스에 집중 가능

AWS Lambda : 대표적인 서버리스 서비스, 불필요한 서버관리 필요 x -> 이중화 기본 제공, 보안, 오토 스케일링 
사용사례 : 웹앱, 백앤드, 챗봇 등등

AWS API Gateway : 사용자에게 엔드포인트 제공하여 요청 받아서 백엔드 전달, REST APIs, HTTP APIs

DynamoDB : 서버리스 NoSQL DB, 서버리스의 장점 보유, ms의 짧은 지연시간 보장, 다수의 샤드로 쪼개 수평 확장 -> 높은 성능 보장
각 Table은 여러 Item 보유 , 하나의 Item (RDBMS의 row)은 여러 개의 Attribute( ==Rdbms의 컬럼 개념), Partition Key로 PK 제공 or PartitionKey + Sort Key로 PK 제공

S3 : 심플 스토리지 서비스, Object 스토리지, 파일 + 파일을 설명하는 메타데이터 = Object , 3곳 이상의 가용 영역에 2 카피씩 저장, 모두 암호화, 세밀한 엑세스 제어를 통한 높은 보안수준 확보, 거의 모든 형태의 데이터 저장 가능 -> 정적 웹사이트 호스팅도 가능
